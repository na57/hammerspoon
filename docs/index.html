项目简介

Hammerspoon 是一款强大的 macOS 自动化工具，允许用户使用 Lua 脚本编写自定义的快捷键和自动化操作。本项目提供了一套精心设计的 Hammerspoon 配置，旨在提高 macOS 用户的工作效率。

通过这套配置，您可以轻松实现多显示器管理、窗口控制、文件管理、鼠标自动化等功能，大幅简化日常操作流程。

项目目标：为 macOS 用户提供一套开箱即用、功能完善、易于定制的 Hammerspoon 配置方案。

核心功能特点

1. 🖥️ 显示器智能管理

功能描述：支持多显示器环境下的窗口快速切换，同时提供显示器残影清除功能，确保显示效果清晰。

使用场景：

  • 在多个显示器之间快速移动窗口
  • 清除显示器残影，提升显示质量
  • 优化多显示器工作流程

快捷键：Hyper + M（切换窗口到下一显示器）、Hyper + G（清除所有显示器残影）

2. 🪟 窗口全屏切换

功能描述：一键切换当前窗口的全屏状态，无需使用鼠标操作。

使用场景：

  • 快速进入专注模式
  • 最大化应用窗口
  • 提高工作专注度

快捷键：Hyper + P

3. ☁️ iCloud 文件快速移动

功能描述：将 Finder 中选中的文件一键移动到 iCloud 指定目录，支持批量移动多个文件。

使用场景：

  • 快速备份文件到 iCloud
  • 整理文件到指定文件夹
  • 批量处理多个文件的移动操作

快捷键：Hyper + 2

使用方法：

  1. 在 Finder 中选中要移动的文件（可多选）
  2. 按下 Hyper + 2
  3. 文件会自动移动到 iCloud 的指定目录

启用与禁用说明

启用功能：

  1. 打开 /Users/na57/workshop/hammerspoon/icloud.lua 文件
  2. 找到 getTargetDirectory() 函数中的路径配置
  3. 修改路径为您需要的目标 iCloud 目录
  4. 保存文件后，按下 Hyper + R 重新加载配置

禁用功能：

  1. 打开 /Users/na57/workshop/hammerspoon/init.lua 文件
  2. 找到以下代码段：
    3. hs.hotkey.bind(hyper, "2", icloud.moveToICloud)
  3. 在该行代码前添加注释符号 --，使其变为：
    4. -- hs.hotkey.bind(hyper, "2", icloud.moveToICloud)
  4. 保存文件后，按下 Hyper + R 重新加载配置使更改生效

4. 🖱️ 鼠标自动移动

功能描述：自动移动鼠标以防止系统休眠，适用于需要长时间运行任务或保持系统活跃的场景。

使用场景：

  • 长时间下载文件时防止休眠
  • 运行需要持续活跃的任务
  • 保持在线状态

快捷键：Hyper + 5（启动）、Hyper + 6（停止）

5. ⚡ 配置快速重载

功能描述：一键重新加载 Hammerspoon 配置，方便在修改配置后立即生效。

使用场景：

  • 修改配置后快速测试
  • 调试配置文件
  • 应用新的快捷键设置

快捷键：Hyper + R

安装步骤

步骤 1：安装 Hammerspoon

  1. 访问 Hammerspoon 官方网站：https://www.hammerspoon.org/
  2. 下载最新版本的 Hammerspoon 应用程序
  3. 将下载的应用程序拖拽到应用程序文件夹
  4. 启动 Hammerspoon 应用程序

步骤 2：配置文件设置

  1. 打开 Hammerspoon 应用程序
  2. 点击菜单栏中的 Hammerspoon 图标
  3. 选择"Preferences"（偏好设置）
  4. 在"Config"选项卡中，点击"Open Config"按钮
  5. 将本项目的配置文件内容复制到打开的配置文件中
  6. 保存配置文件

步骤 3：加载配置

  1. 在 Hammerspoon 菜单栏图标上点击
  2. 选择"Reload Config"（重新加载配置）
  3. 或者使用快捷键 Hyper + R
  4. 如果配置成功，您将看到一个提示消息

使用方法

基本操作流程

在使用本配置之前，请确保您已经完成了安装步骤。配置加载成功后，您可以使用以下快捷键：

  • Hyper 键：默认为 Cmd + Shift + Option，您可以在配置文件中自定义
  • 快捷键组合：按住 Hyper 键 + 对应的字母键或数字键即可触发功能

常见任务实现方式

任务 1：在多个显示器之间移动窗口

  1. 确保您已连接多个显示器
  2. 激活要移动的窗口
  3. 按下 Hyper + M
  4. 窗口将自动移动到下一个显示器

注意：此功能支持全屏窗口的移动。

任务 2：清除显示器残影

  1. 按下 Hyper + G
  2. 系统将自动清除所有显示器的残影
  3. 清除过程可能需要几秒钟时间

注意：此功能会刷新显示器，可能会导致短暂的屏幕闪烁。

任务 3：将文件移动到 iCloud

  1. 在 Finder 中选中要移动的文件（可多选）
  2. 按下 Hyper + 2
  3. 文件将自动移动到配置的 iCloud 目录
  4. 您将收到移动成功的提示

注意：确保您已在配置文件中设置了正确的 iCloud 目标目录。

任务 4：防止电脑休眠

  1. 按下 Hyper + 5 启动鼠标自动移动
  2. 鼠标将每 30 秒自动移动一次
  3. 系统将保持活跃状态，不会进入休眠
  4. 按下 Hyper + 6 停止鼠标移动

注意：长时间使用此功能可能会影响鼠标的使用体验。

任务 5：切换窗口全屏

  1. 激活要切换全屏状态的窗口
  2. 按下 Hyper + P
  3. 窗口将自动切换全屏状态
  4. 再次按下可退出全屏

注意：此功能适用于大多数应用程序窗口。

任务 6：重新加载配置

  1. 修改配置文件后
  2. 按下 Hyper + R
  3. 配置将立即重新加载
  4. 您将看到配置加载成功的提示

注意：重新加载配置不会影响正在运行的应用程序。

常见问题解答（FAQ）

1. 快捷键按下后没有反应

可能原因：

  • Hammerspoon 未正确启动
  • 配置文件未正确加载
  • 快捷键与其他应用程序冲突

解决方法：

  1. 检查 Hammerspoon 是否在菜单栏中运行
  2. 重新加载配置文件（Hyper + R）
  3. 检查是否有其他应用程序占用了相同的快捷键
  4. 查看 Hammerspoon 控制台是否有错误信息

2. 如何修改快捷键

修改步骤：

  1. 打开 Hammerspoon 配置文件（~/.hammerspoon/init.lua）
  2. 找到对应的快捷键绑定代码
  3. 修改 hs.hotkey.bind() 函数中的第二个参数（字母键）
  4. 保存文件并重新加载配置

示例：

hs.hotkey.bind(hyper, "M", display.moveWindowToNextScreen)

将 "M" 改为其他字母键即可。

3. 如何修改 Hyper 键

修改步骤：

  1. 打开配置文件
  2. 找到 Hyper 键的定义代码
  3. 修改键位组合
  4. 保存并重新加载配置

示例：

local hyper = {"cmd", "shift", "alt"}

可以改为 {"cmd", "alt"} 或 {"cmd", "ctrl"} 等。

4. iCloud 文件移动失败

可能原因：

  • 目标目录不存在
  • 文件权限不足
  • iCloud 同步未启用

解决方法：

  1. 检查配置文件中的目标目录路径是否正确
  2. 确保目标目录存在且可写
  3. 检查 iCloud 同步是否正常工作
  4. 查看 Hammerspoon 控制台的错误信息

5. 鼠标自动移动功能的影响

功能说明：

鼠标自动移动功能会定期移动鼠标光标，以防止系统休眠。这可能会影响您在使用鼠标时的体验。

注意事项：

  • 长时间使用可能会干扰正常操作
  • 建议在需要时启动，不需要时及时停止
  • 可以使用 Hyper + 6 随时停止

6. 如何卸载或禁用某个功能

禁用功能：

  1. 打开配置文件
  2. 找到要禁用的功能对应的快捷键绑定代码
  3. 在代码前添加注释符号 --
  4. 保存并重新加载配置

示例：

-- hs.hotkey.bind(hyper, "M", display.moveWindowToNextScreen)

7. Hammerspoon 对系统性能的影响

性能影响：

Hammerspoon 本身对系统性能的影响很小，但在以下情况下可能会有轻微影响：

  • 配置了大量的快捷键和自动化脚本
  • 使用了频繁执行的定时任务
  • 同时运行多个复杂的脚本

优化建议：

  • 只启用需要的功能
  • 避免使用过于复杂的脚本
  • 定期检查和优化配置

8. 如何查看错误日志

查看方法：

  1. 点击菜单栏中的 Hammerspoon 图标
  2. 选择"Console"（控制台）
  3. 在控制台中可以查看所有的日志和错误信息
  4. 错误信息会以红色显示

常见错误：

  • 配置文件语法错误
  • 模块加载失败
  • API 调用错误

9. 是否支持多显示器

支持情况：

本配置完全支持多显示器环境，包括以下功能：

  • 在多个显示器之间移动窗口
  • 清除所有显示器的残影
  • 独立控制每个显示器

使用建议：

  • 确保所有显示器都已正确连接
  • 在系统偏好设置中配置显示器排列
  • 测试每个功能在不同显示器上的表现

10. 如何备份和恢复配置

备份配置：

  1. 打开配置文件目录（~/.hammerspoon/）
  2. 复制所有配置文件到备份位置
  3. 建议定期备份，特别是在修改配置前

恢复配置：

  1. 将备份的配置文件复制回 ~/.hammerspoon/ 目录
  2. 重新加载 Hammerspoon 配置
  3. 验证所有功能是否正常工作

版本控制：

建议使用 Git 等版本控制工具管理配置文件，方便追踪修改历史和回滚。
